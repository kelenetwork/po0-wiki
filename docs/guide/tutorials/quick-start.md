# 从购买到连通

这是一条最短路径：先让出口可用，再让 Po0 转发，最后把客户端配置改成连接 Po0。

## 第一步：确认你要买哪类产品

如果你需要国内入口，一般先看广州 BGP 或华东 BGP。  
如果你需要香港/美国位置本身，则看 T1 香港 / 美国。

购买前先读：

- [Po0 适合谁](/guide/buying/who-is-po0-for)
- [入口地区怎么选](/guide/buying/choose-entry-region)
- [使用前需要确认的限制](/guide/notes/pre-use-limitations)

## 第二步：准备出口或落地机

常见做法是先在出口机上搭好代理协议，并确认它自己能用。

你至少需要记录：

- 出口机 IP；
- 出口机协议端口；
- 客户端配置链接；
- 出口机所在地区。

如果出口机直连都不可用，先不要配置 Po0，避免把问题混在一起。

## 第三步：在 Po0 上配置端口转发

在 Po0 上选择一个监听端口，把它转发到出口机 IP 和端口。

配置完成后，链路应该变成：

```text
客户端 → Po0 公网 IP:监听端口 → 出口机 IP:协议端口
```

社区教程里常见做法是用 nftables 管理转发规则，也有交互式脚本可以自动安装 nftables、开启 IPv4 转发、增删转发端口。如果想用菜单脚本，见：[nftables 转发脚本配置](/guide/tutorials/nftables-script)。如果想手动维护规则，见：[nftables 手动转发配置](/guide/tutorials/nftables-port-forwarding)。

## 第四步：替换客户端配置

把原本客户端配置里的：

```text
出口机 IP:出口机端口
```

替换成：

```text
Po0 公网 IP:Po0 监听端口
```

协议、密码、加密方式等保持不变。你只是把“连接入口”从出口机换成 Po0。

## 第五步：测试

先做三类测试：

1. 客户端是否能连通；
2. 晚高峰是否稳定；
3. [Looking Glass](/looking-glass) 中对应方向的延迟和路由是否符合预期。

能连通只是第一步，长期体验还要看稳定性和丢包。
