# 故障反馈应该提供哪些信息

## 用途

帮助用户在遇到访问异常时一次性提供足够信息，让支持人员更快定位问题范围和可能原因。

## 适合人群/场景

- 访问变慢、不可达或体验波动，需要提交反馈的用户。
- 团队内部需要统一故障反馈格式的客户。
- 希望减少来回追问、提高处理效率的用户。

## 文章大纲

1. 反馈前先确认是否为持续问题
2. 需要提供的基础信息
3. 需要提供的使用环境信息
4. 建议提供的测试结果
5. 如何描述影响范围和发生时间
6. 一个完整反馈示例
