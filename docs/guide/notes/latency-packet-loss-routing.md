# 怎么看延迟、丢包和路由波动

## 用途

用面向用户的方式解释延迟、丢包和路由波动的基本含义，帮助用户判断哪些情况需要关注，哪些属于正常波动。

## 适合人群/场景

- 使用中发现访问变慢、卡顿或偶发不可达的用户。
- 需要阅读 Looking Glass 或其他测试结果的客户。
- 希望区分本地网络、目标站点和服务状态影响的用户。

## 文章大纲

1. 延迟代表什么，不能代表什么
2. 丢包应该看持续性和影响范围
3. 路由波动为什么会发生
4. 单点异常和大范围异常如何区分
5. 什么时候需要继续观察
6. 什么时候应该提交反馈
