# 使用前需要确认的限制

## 用途

帮助用户在购买和使用前确认可能影响体验的限制条件，避免购买后发现与预期不一致。

## 适合人群/场景

- 准备购买 Po0，想提前确认服务边界的用户。
- 有固定业务、固定目标或特殊使用环境的客户。
- 对稳定性、可用性或访问范围有明确要求的团队。

## 文章大纲

1. 确认主要使用地区是否合适
2. 确认主要访问目标是否合适
3. 确认设备、客户端和网络环境要求
4. 确认预算与稳定性预期是否匹配
5. 确认是否需要长期观察或试用验证
6. 购买前建议向客服说明的信息
